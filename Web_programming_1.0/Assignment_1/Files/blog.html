What is robots.txt ?

A robots.txt file is a file at the root of your site that indicates those parts of your site you don’t want accessed by search engine crawlers. The file uses the Robots Exclusion Standard, which is a protocol with a small set of commands that can be used to indicate access to your site by section and by specific kinds of web crawlers (such as mobile crawlers vs desktop crawlers). The two main directives of the standard are:
- User-agent : Defines the search engine that a rule applies to.
- Disallow : Advises a search engine not to crawl and index a file, page, or directory

How have you configure it for your site ?

User-agent: *
Disallow: https://1dv525.github.io/cf222jb-examination-1/Files/contact.html
An asterisk (*) can be used as a wildcard with User-agent to refer to all search engines.
Advises a search engine not to crawl and index a file, page, or directory. Here for example, I used disallow to not craws and index the contact.html file.

What is human.txt ?

It's an initiative for knowing the people behind a website. It's a TXT file that contains information about the different people who have contributed to building the website. By adding a txt file, you can prove your authorship (not your property) in an external, fast, easy and accessible way. The internet is for humans. The only aim of this initiative is to know who the authors of the sites we visit are.

How have you configure it for your site ?

/* TEAM */
It's information about me, where I founded the website.
/* SITE */
Information about the website, last update, how realised.
/* SOURCE */
Website where I founded help for building the website/

What is Open Graph ?

The Open Graph protocol enables any web page to become a rich object in a social graph. For instance, this is used on Facebook to allow any web page to have the same functionality as any other object on Facebook (or twitter).
While many different technologies and schemas exist and could be combined together, there isn't a single technology which provides enough information to richly represent any web page within the social graph. The Open Graph protocol builds on these existing technologies and gives developers one thing to implement. Developer simplicity is a key goal of the Open Graph protocol which has informed many of the technical design decisions.

How do you make use of it ?

Using the protocol provide you accurate information that search engines need. This can raise the rank of your pages. This can relocate your website to higher positions in search engines like Google, because Google detects Open Graph meta tags. Finally, the social media are taking so much importance on the web, so it's really important to be available on the social media. That's why Open Graph is so important.

Why is it important to separate the layout and design (css) from content (html)?

Here are the most compelling reasons for using CSS and HTML over outdated methods:

  • Efficient of code : The larger your files are, the longer they will take to download.
  • Ease of maintenance : Following on from the last point, if your styling and layout information is only specified in one place, it means you only have to make updates in one place if you want to change your site’s appearance.
  • Device compatibility : Because my HTML page is just plain markup, with no style information, it can be reformatted for different devices with vastly differing attributes (eg screen size) by simply applying an alternative style sheet — you can do this in a few different ways
  • Web crawlers/search engines : Chances are you will want your pages to be easy to find by searching on Google, or other search engines.
  • My point of view : If you talk to any professional standards-aware web developer or designer, and they’ll tell you that separating content, style, and behaviour is the best way to develop a web application.
